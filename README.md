## 🎨 核心设计理念 & UI 展现逻辑

本小站的核心展现逻辑与设计理念包含以下几大支柱：

### 1. 每日数据更新与智能主力合约筛查机制 (Watchlist Filter Logic)
本站定位为**按日更新的动态量化分析页面**。每日收盘后（15:00 及夜盘结束后），后台通过运行 `sync_data.py` 脚本，全量扫描国内 5 大期货交易所所有 69 个国内商品期货主力合约。

主力合约的筛选与异动警示规则**严格基于持仓量 (Open Interest, OI) 历史周期和上市年限进行多维度过滤**：
* **筛选合约基准**：筛选逻辑始终基于持仓量最大的具体月份主力合约（例如 `SP2609`、`CU2608`）。在前端看板展示时，以 `纸浆主力(SP2609)` 的直观结构呈现。
* **上市时间 ≥ 3年**：
  - 若当前主力合约的持仓量（OI）创出过去 10 年单日历史最高峰值，则触发 **`OI创新高`** 标签。
  - 若当前主力合约持仓量达到过去 10 年单日最高持仓峰值的 **90% 以上**（`oiRatio >= 0.9`），则触发 **`OI近历史高`** 标签。
  - 触发上述任一标签的合约，均定义为“异动合约”，并在市场看板顶部的自选滚动栏中进行高亮渲染。
* **1年 ≤ 上市时间 < 3年**：
  - 剔除“近历史高”判断，**必须且仅在主力合约持仓量创出历史新高**（超过过去 10 年最高单日持仓量峰值）时，才触发异动并进入市场看板。
* **上市时间 < 1年**：
  - 不进入任何筛选机制，**不触发任何异动标签，无法进入市场看板**，规避新合约缺乏历史极值参考导致的噪声干扰。

### 2. 筹码分布双直方图第一原理 (Market Profile Construction Rules)
在市场看板主图表中，行情展示 K 线支持多周期频率（15m / 30m / 1H / 日K / 周K）切换，叠加 MA5/10/20 和 VOL 交易量。最核心的可视化技术为双向叠加的 **TPO Profile (时间价格机会图)** 和 **Volume Profile (成交量分布图)**，其底层的构建算法根据数据精度分级执行：
* **TPO Profile (30m / 日 / 周)**：
  - **构建规则**：**严格基于 30分钟（30m）K线** 抓取的 OHLC 和 Volume 数据进行构建。
  - **核心计算逻辑 (`profile.js` - `buildTpoProfile`)**：
    1. 根据不同品种的基准跳动点（Tick Size）在当前价格区间内分箱（Bins）；
    2. 将每一个 30m 周期作为时间 print 代表，在对应的价格 Bin 中累加时间机会频数；
    3. 寻找累加机会最高的 Bin 作为 **POC（筹码控制中心）**，并向外双向扩展计算覆盖 70% 交易时间的 **VAH/VAL（价值区间上下沿）**，在 Canvas 左侧渲染出横向直方图。
* **Volume Profile (30m / 日 / 周)**：
  - **构建规则**：**严格基于 1分钟（1m）K线** 抓取的价格与成交量数据进行构建。
  - **降级与极限逻辑 (`chart.js` - `Volume Profile Fallback`)**：
    1. 前端首选读取 1 分钟精度数据源（`bars1m`）构建 Volume Profile；
    2. **降级逻辑**：若由于数据源时间段限制，1m 细分数据已达抓取极限，系统将自动降级，使用 **5分钟（5m）K线** 数据（`bars5m`）进行替代构建；
    3. **极限逻辑**：若 5m 精度数据也达到抓取极限，系统将**放弃并停止构建 Volume Profile**；
    4. **严格限制**：**5m 及以上频率（如 15m、30m、日线）的数据绝不作为构建 Volume Profile 的标准**，确保筹码分布图的高精度与真实性。
  - **核心计算逻辑 (`profile.js` - `buildVolumeProfile`)**：通过均匀分配算法将 1m/5m 的 Volume 分摊到各价格 Bin，计算 POC、70% 价值区间（VAH/VAL）并在右侧渲染直方图，同时提取高/低量能节点（HVN/LVN）辅助研判。

## 📊 页面功能模块 & 后期规划

全站共包含四个核心功能模块，通过顶部毛玻璃导航条进行平滑切换：

### 1. 市场行情看板 (Market Dashboard)
* **顶部滚动看板**：展示触发 OI 异动的自选/异常品种卡片。激活卡片有边框发光动效，联动渲染整版行情。
* **Canvas 主图**：支持 Candle/Line 切换，叠加 MA5/10/20，下方展示 VOL 副图。TPO（左）与 Volume Profile（右）双向半透明覆盖，带有十字光标和详细数据 Tooltip。
* **侧边量化小组件**：期限结构 Basis 曲线展示（评估市场处于 Contango 升水还是 Backwardation 贴水），CTA 均线信号（看多/看空/中性），以及基于乘数与保证金比例测算的单手交易保证金。

### 2. 投资思考 (Investment Thoughts)
* 瀑布流展示宏观、商品和配置的研究文章。点击文章原地毛玻璃滑出全屏阅读器，附有顶部进度条。
* **🔮 后期规划**：未来将在此模块中，加入针对所有交易所主要活跃合约的**“交割品企业完税极限成本计算框架与多维度流程”**。提供详实的企业完税极限模型、关税/运费/资金利息损耗等全维度参数配置及交割品理论成本计算方法。

### 3. 技术异动监控 (Technical Anomaly Monitor)
* **全市场持仓排名概览**：以列表形式展示所有交易所全部 69 个商品期货的主力合约。展示当前持仓量相对于 10 年历史单边最高峰值的百分比并进行降序排列。
* **🔮 后期规划**：未来在此页面中加入**“主力合约价格与该合约对应交割商品的企业完税极限成本价的差值比较排名”**。对比期货实际价格与其交割商品在企业完税极限下生成的成本价差，作为价值发现和估值偏离的重要监控指标。

### 4. 关于本站 (About Me)
* 记录个人“安全边际与风险暴露控制”投资哲学，以及按日更新的后台数据同步机制。

## 🚀 部署与运行

### 行情模拟演示模式
如果直接双击 `index.html` 运行（受浏览器 CORS 跨域限制，无法直接 Fetch 本地 JSON），小站会自动进入“演示模式”，启动实时 Tick 模拟时钟，生成仿真 K 线与持仓变动以供系统演示。

### 真实数据运行模式 (推荐)
要在本地查看最新同步的真实国内期货行情，请使用本地静态 HTTP 服务器：

1. **安装 Python 依赖**：
   ```bash
   pip install akshare pandas
   ```
2. **运行同步脚本**：
   ```bash
   python sync_data.py
   ```
   *该脚本会自动全量抓取 69 个国内商品期货合约的最新行情。在同步过程中，系统会自动处理历史持仓量的口径变更：由于新浪数据库在 **2020-01-02** 切换了持仓量口径（2020年前为双边，2020年后为单边），脚本自动将 2020 年之前的历史持仓量除以 `2.0`，从而杜绝了历史极值虚高的问题。*
3. **启动本地服务器**：
   直接双击运行 [start_local.bat](file://c:/Users/78432/OneDrive/桌面/personal-investment-site/start_local.bat) 或在命令行中运行：
   ```bash
   python server.py
   ```
   服务器会自动寻找空闲端口，并在默认浏览器中为您打开 http://localhost:8000。此时，顶部状态指示灯会转为蓝色：**全市场持仓异动扫描 ✓**。

## 🛠 技术框架与设计细节

- **排版与样式**: 使用 Vanilla CSS 和全局 CSS 变量搭建了一整套精致、高对比度的深浅主题系统。
- **微动画与模糊**: 大量应用 `backdrop-filter: blur(...)` 毛玻璃质感、卡片上浮悬停阴影、平滑路由切换。
- **自研 Canvas 图表**: 行情图表（`chart.js`）完全采用 HTML5 Canvas 直接绘制蜡烛线、价格坐标、均线、成交量以及筹码直方图，加载速度极快、内存占用低，且具备完全的可定制性。
- **高性能筹码分布计算**: 由原生 JavaScript（`profile.js`）在前端高效处理 TPO 与 Volume Profile 的分箱、VAH/VAL 计算以及 POC 检测，无任何外部库依赖。
- **矢量图标**: 引入了现代极简的 Lucide 矢量图标库，提升视觉高级感。
