# 山风蛊小站 | Summit Wind Portal

在线 UI 链接：[https://kannzz3.github.io/personal-investment-site/](https://kannzz3.github.io/personal-investment-site/)

山风蛊小站是一个个人投资研究与中国商品期货行情可视化页面。当前版本围绕“全市场持仓量扫描 -> 筛选合约异动 -> 展示异动合约高精度图表”运行，前端读取 `data/futures_data.json`，不在浏览器内直接连接交易所或实时行情接口。

## 当前 UI 模块

### 市场看板

- 顶部展示当前筛选出的异动合约卡片，并联动主图。
- 主图支持 K 线与分时切换。
- K 线周期支持 `15M`、`30M`、`1H`、`4H`、`日K`、`周K`、`月K`。
- `4H` 由已抓取的 `1H/min60` 数据在前端聚合得到，不额外抓取。
- `周K`、`月K` 由当前具体合约的日线数据在前端聚合得到。
- 指标支持 `MA5`、`MA10`、`MA20` 与 `VOL`。
- 图表支持十字光标、缩放滚动、全屏、水平线与折线绘制。

### 投资思考

- 展示站内投资研究文章卡片。
- 支持按交易所和随笔分类过滤。
- 点击文章后在页面内打开阅读器，显示作者、日期、阅读时间与阅读进度。

### 合约异动

- 页面标题为“合约异动监控”。
- 展示全市场持仓量排名概览，包含品种、交易所、当前持仓、历史峰值、峰值日期、OI 比率与状态。
- 筛选条件：
  - `OI创新高`：当前持仓量高于历史 10 年单日峰值。
  - `OI近历史高`：当前持仓量达到历史 10 年单日峰值的 90% 以上。
- 页面说明文案为“全市场持仓量扫描 · 历史峰值阈值触发 · 结构性异动标的分析”。

### 关于本站

当前 UI 中“关于本站”说明了四条核心逻辑：

1. 异动合约筛选：基于合约上市年限与 OI Ratio 做差异化过滤。
2. 异动合约图表展示：针对筛选出的异动合约展示高精度筹码图表。
3. 投资思考：沉淀大宗商品研究与资产配置思考。
4. 合约异动：每日追踪各交易所合约持仓水平与估值偏离。

页面保留 GitHub 主页链接和邮件联系入口。

## 数据同步逻辑

核心同步脚本为 `sync_data.py`，输出文件为 `data/futures_data.json`。

当前同步流程分两段：

1. 全市场扫描
   - 使用新浪财经数据扫描 `SHFE`、`DCE`、`CZCE`、`INE`、`GFEX` 覆盖的 69 个商品期货品种。
   - 对每个品种计算当前主力持仓、10 年历史峰值、峰值日期、OI 比率与异动状态。
   - 只有达到 `near_high` 或 `new_high` 的品种会进入异动列表。

2. 异动合约详情抓取
   - 只对异动合约抓取详细图表数据。
   - 优先使用天勤 `TqSdk` 解析主连当前对应的具体主力合约，并抓取该具体合约的分钟 K 线。
   - 若没有天勤账号密码或认证失败，自动降级使用新浪财经接口抓取可用分钟 K 线。
   - 异动合约日线使用具体合约日线，例如 `SP2609`，不再使用连续主连日线；周线和月线继续由该具体合约日线聚合。

导出的字段包含：

- `metadata.sync_time`：同步时间。
- `metadata.anomalies`：当前异动品种列表。
- `metadata.screening`：全市场扫描结果。
- `metadata.contracts`：合约元数据、交易所、保证金、乘数、持仓分析等。
- 各品种顶层数据：`daily`、`min1`、`min5`、`min15`、`min30`、`min60`。

非异动品种主要保留行情和筛选元数据；异动品种才写入详细 K 线数据，以控制 JSON 体积和页面加载速度。

## 图表与 Profile 规则

### K 线数据

- 分钟数据：`1m`、`5m`、`15m`、`30m`、`60m` 最多抓取约 1500 根。
- `4H`：由 `60m` 数据按交易日内顺序每 4 根聚合。
- 日线：异动合约使用当前具体合约日线。
- 周线、月线：由当前具体合约日线在前端聚合。

同步脚本会过滤无效 K 线，并使用 `json.dump(..., allow_nan=False)`，避免 `NaN` 写入 JSON 导致浏览器无法解析。

### TPO Profile

- UI 支持 `30m TPO`、`日 TPO`、`周 TPO`。
- TPO 构建严格基于 `30m` K 线数据。
- 前端根据 30 分钟 Bar 进行价格分箱、POC、VAH、VAL 计算并绘制。

### Volume Profile

- UI 支持 `30m VP`、`日 VP`、`周 VP`。
- VP 优先基于 `1m` 数据构建。
- 当 `1m` 数据不可用或超出可用范围时，降级使用 `5m` 数据。
- 若 `5m` 也不可用，则停止构建 VP。
- 不使用 `15m`、`30m`、`60m` 或日线作为 VP 构建源。

## GitHub Actions 自动更新

工作流文件：`.github/workflows/sync_data.yml`

触发方式：

- 工作日 `08:00 UTC`，对应北京时间 `16:00`。
- 工作日 `19:00 UTC`，对应北京时间次日 `03:00`。
- 支持在 GitHub Actions 页面手动触发。

工作流会：

1. 安装 Python 3.11。
2. 安装 `akshare`、`pandas`、`tqsdk`。
3. 通过仓库 Secrets 注入 `TQ_USERNAME` 和 `TQ_PASSWORD`。
4. 运行 `python sync_data.py`。
5. 若 `data/futures_data.json` 有变化，自动提交并推送。

如需启用云端天勤高精度模式，请在 GitHub 仓库中配置：

- `TQ_USERNAME`
- `TQ_PASSWORD`

如果未配置或认证失败，脚本会自动降级到新浪财经备用接口。

## 本地运行

安装依赖：

```bash
pip install akshare pandas tqsdk
```

本地同步数据：

```bash
python sync_data.py
```

启动本地服务：

```bash
python server.py
```

也可以运行：

```bash
start_local.bat
```

直接双击 `index.html` 时，浏览器可能因本地文件跨域限制无法读取 JSON；推荐通过 `server.py` 启动本地 HTTP 服务。

## 文件结构

- `index.html`：页面结构与四个主导航模块。
- `style.css`：主题、布局、响应式与组件样式。
- `app.js`：前端状态管理、数据加载、UI 渲染、周期聚合与交互。
- `chart.js`：Canvas K 线、指标、绘图工具、TPO/VP 叠加渲染。
- `profile.js`：TPO Profile 与 Volume Profile 的计算逻辑。
- `sync_data.py`：全市场扫描、异动筛选、详细行情抓取与 JSON 导出。
- `.github/workflows/sync_data.yml`：云端自动同步工作流。
- `data/futures_data.json`：网页读取的行情与筛选结果数据。
